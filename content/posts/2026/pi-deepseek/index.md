先说结论：**Pi + DeepSeek，是我目前找到的最具性价比的 AI 编码备用方案。**

两毛四，写了一篇完整的博客文章。

---

## 为什么需要备用方案

我同时在用 Claude Code、Codex、Copilot。每月加起来几十美元，不算便宜，但还能接受。

真正的问题是**额度限制**。

不是用完即停的硬限制——是那种用着用着就感觉它在"省着花"的软限制。高峰期的体验明显降级，付费了，却没得到完整的服务。

我不想花上百美元去升额度。我需要的是一个随时能顶上、好用还不贵的备用方案。

---

## 试了 OpenCode，没跑起来

第一个试的是 OpenCode。最近讨论度很高。

但在我的 Mac 上直接打不开。界面都没进去。后来试着给它配 DeepSeek API，流程也不顺畅，折腾一圈放弃了。

这时候想起了 Pi。

---

## Pi 是什么

[Pi][1] 是一个终端 AI 编码代理，由 [@badlogic][1] 开发。开源，自带 API Key，不锁定任何模型。

Pi 只提供终端界面，所有交互都在命令行里完成。这正好适合我这种喜欢命令行工具的人。

最近 DeepSeek-V4 刚发布，价格大概是 GPT-5.5 的十分之一——正好接进 Pi，试试效果。

---

## 实战：让它写一篇文章

正好我最近给 Explain Error Plugin 加了不少新功能，需要写文章分享。

我没给任何详细 prompt。就一句话：

> "Explain Error 最近发布了一些新功能，自从我上次写完文章后，这段时间的功能都没分享过，你帮我把这些新功能整理一下，写一篇文章。"

然后我就不管了。

它自己干了几件事：去 GitHub 拉了 closed issues，把 11 项更新按"重大功能 / 用量管控 / 新 Provider / Bug 修复"四个维度分组，提取每个功能的技术细节和配置方式，连我博客惯用的表格格式和脚注风格都还原了。

整个过程，我没插过一个字。

最终费用：**¥0.24**。

---

## 值在哪

第一，**成本透明**。每次调用花多少钱，清清楚楚。不会月底发现账单爆了。

第二，**没有软限制**。API 直连无所谓额度，用多少花多少，不做限速。

第三，**模型自由**。DeepSeek 写代码，Qwen 分析中文长文本，想用哪个换哪个。Codex 只给自家模型。

第四，**开源**。Pi 在 [badlogic/pi-mono][1]，社区活跃，支持 TypeScript 扩展和 Pi Packages——你甚至可以自己打包分享 Skills。

---

## 不完美的地方

得说清楚。

配置不是开箱即用——你要自己去 DeepSeek 申 API Key、充值。不难，但多一步。

Pi 是终端工具，不是图形 IDE。习惯了鼠标操作的人，头几天会不适应。它走的是"代理"路线，在终端里读写代码仓库，而不是在你敲代码时逐行补全。

所以我并没有放弃 Codex。主力还是它——已经付过费了。当额度吃紧或者需要跑批量任务（写文章、重构、生成测试），切到 Pi + DeepSeek。

这不是替代，是互补。
